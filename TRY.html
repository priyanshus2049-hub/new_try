Hackathon Strategy & API Bridge

The "Bridge" You Need to Build

The APIs provide raw data. The "Bridge" is the Similarity Algorithm. FlavorDB gives you the molecules of Food A. You must write logic (or finding the specific endpoint) to find Food B that shares those molecules.

Logic flow for your backend:
1. GET /entities?name=Peanut -> Get ID.
2. GET /entities/{id}/molecules -> Get Molecule List A.
3. (The Bridge) Iterate other foods OR use /entities/{id}/similar (if available and working) to find overlapping lists.
4. Filter out allergens (e.g., if input is Peanut, remove Legumes).

FlavorDB Endpoints to Use

  • Search: {{baseUrl}}/entities?name={query} (To get ID)
  • Composition: {{baseUrl}}/entities/{id}/molecules (To get chemical data)
  • Golden Endpoint: {{baseUrl}}/food/by-alias?food_pair={name} (Try this for pairing/similarity)

RecipeDB Endpoints to Use

  • Search by Ingredient: {{baseUrl}}/recipes/search?ingredient={substitute}
  • Get Recipe Details: {{baseUrl}}/recipe/{id}
